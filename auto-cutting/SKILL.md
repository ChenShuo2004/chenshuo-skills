# 自动剪辑

## 目标

使用这个 skill，把自动剪辑需求转成可验证、可维护的工程流程，并在用户需要时直接生成视频成片。它负责自动剪辑的业务逻辑、渲染路径与验收规则；如果用户要把需求交给 Ralph 执行，再配合 `ralph-runner` 使用。

## `$` 快捷触发

- `$auto-cutting`：通用自动剪辑入口，先判断输出模式，再执行对应流程。
- `$auto-cutting-prd`：只做需求文档、剪辑方案或 Ralph PRD。
- `$auto-render-video`：直接剪辑、渲染并验证 mp4 成片。
- `$auto-cutting-ralph`：自动剪辑需求到 Ralph PRD、dry-run 和迭代反馈的完整闭环。
- `$ralph-runner`：通用 Ralph PRD 与构建执行器。

## 开始前先阅读

修改代码前，先阅读目标项目里的需求文档、README、产品说明、接口说明、现有草稿样例和输出样例。如果需求文档与代码实现不一致，先说明差异，再决定怎么改。

如果用户还没有需求文档，使用 `assets/requirements-template.md` 创建一份，建议放在目标项目中：

```text
docs/requirements/auto-cutting.md
```

## 先确定输出模式

实现前必须先确定一种输出模式：

- **只生成方案**：输出时间线计划 JSON 或 Markdown 剪辑决策清单。
- **生成剪映/CapCut 草稿**：生成新草稿，或复制旧草稿后修改；默认不覆盖用户原草稿。
- **直接渲染视频**：当用户要直接成片时使用；优先使用项目已有 FFmpeg、MoviePy、Remotion 等确定性渲染链路，没有时可使用本 skill 的 `scripts/render_plan_ffmpeg.py`。
- **GUI 自动化**：只有用户明确接受脆弱性时才使用；需要截图、日志和人工兜底步骤。

如果输出模式不明确，先说明假设，并采用最安全的模式：只生成方案或生成新草稿。用户明确要求“直接剪辑视频、直接成片、导出 mp4”时，默认选择直接渲染视频。

## 直接剪辑视频

当用户要求直接剪辑并导出视频时，先生成或读取结构化渲染计划。可从 `assets/render-plan-template.json` 复制一份到目标项目输出目录，再填入素材路径、入点、出点、目标画幅和输出文件。

如果目标项目没有自己的渲染脚本，使用内置 FFmpeg 脚本：

```text
py -3 scripts\render_plan_ffmpeg.py path\to\render-plan.json
```

脚本依赖本机可用的 FFmpeg 和 FFprobe。脚本支持：视频片段裁剪、顺序拼接、统一分辨率/帧率/画幅留黑边、无音轨片段补静音、可选 BGM 混音、可选 SRT 字幕烧录、生成验证报告。它不修改原始素材，只写入计划中指定的新输出位置。

复杂剪映专属效果、模板工程、花字或强依赖剪映素材库的需求，优先生成剪映/CapCut 草稿；只有用户接受 GUI 自动化脆弱性时才操作界面。

## 核心流程

1. 明确用户目标：平台、受众、目标时长、画幅比例、风格、节奏、语言和必需素材。
2. 盘点输入：视频、图片、音频、脚本、字幕、品牌素材、现有草稿、输出目录。
3. 提取元数据：时长、分辨率、帧率、音轨、场景边界、是否有转写文本。
4. 生成剪辑计划：片段选择、顺序、入点/出点、字幕、BGM、音量闪避、转场、叠加层、封面帧。
5. 校验计划：路径存在、时长为正、时间线没有非预期重叠或空洞、字幕适配画幅、总时长在容差内。
6. 生成输出：根据输出模式写入新草稿、渲染成片、方案 JSON 或 Ralph story。
7. 验证结果：运行项目测试、草稿校验、渲染冒烟检查，并生成简短验证报告。

更细的业务规则见 `references/workflow.md`。

## 与 Ralph 交接

如果要配合 `ralph-runner`，先创建或更新 Markdown 需求文档，再读取 `references/optimized-ralph-workflow.md`，并参考 `assets/ralph-prd-template.json` 拆成 Ralph stories。

推荐按这些故事拆分：

- 需求文档与输出模式确认。
- 素材扫描与 manifest 生成。
- 媒体元数据提取。
- 时间线规划与剪辑决策模型。
- 草稿写入或视频渲染。
- 字幕、BGM、转场和特效处理。
- 校验报告与失败提示。

每个 story 都要能独立验证。默认通过 `ralph-runner` 做 dry-run 和 no-commit 构建。

## 安全规则

- 不修改原始素材。
- 默认不覆盖已有剪映/CapCut 草稿，除非用户明确要求。
- 生成草稿和中间文件写入带名称或时间戳的新输出目录。
- 保留 manifest，记录所有输入文件、生成文件和校验结果。
- 不随意新增依赖；只有项目已有或媒体探测/渲染确实需要时才新增。
- 涉及删除、清理、覆盖时，先说明准确路径并确认。

## 验证清单

优先使用项目已有命令。至少验证：

- 需求文档存在，并包含输入、输出、输出模式。
- 生成的方案或 PRD 在需要时是合法 JSON。
- 所有引用的素材路径都存在。
- 时间线总时长符合目标容差。
- 没有负时长或零时长片段。
- 草稿或视频输出到了新位置。
- 直接渲染视频时，输出 mp4 文件存在、大小非零，并生成可解析的验证报告。
- 已运行相关测试、类型检查、构建或草稿校验命令。

## 汇报格式

完成后用中文按以下结构汇报：

- 需求理解
- 实现方案
- 关键逻辑
- 修改文件
- 验证方式
- 风险与待确认
