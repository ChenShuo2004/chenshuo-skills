# 自动剪辑 + Ralph 优化流程

## 目标

把自动剪辑需求从“想法”推进到“可执行、可验证、可迭代”的 Ralph 工作流。流程同时覆盖两种产物：

- 需求与方案：Markdown 需求文档、剪辑计划 JSON、Ralph PRD。
- 直接成片：渲染计划、剪映/CapCut 草稿或 mp4 视频、验证报告。

## 分阶段流程

1. 需求收敛
   - 读取用户已有需求、素材说明、脚本、样片、草稿示例和项目 README。
   - 如果没有需求文档，先用 `assets/requirements-template.md` 生成 `docs/requirements/auto-cutting.md`。
   - 必须确定输出模式：计划 JSON、剪映/CapCut 草稿、直接渲染视频、GUI 自动化。

2. 输入盘点
   - 扫描素材目录，生成 manifest。
   - 记录视频、图片、音频、字幕、脚本、BGM、封面、已有草稿路径。
   - 缺失素材必须进入待确认项，不要在 Ralph story 里假装存在。

3. 元数据探测
   - 提取时长、分辨率、帧率、旋转信息、音轨、字幕状态。
   - Windows 路径必须保留原始绝对路径，避免中文、空格、反斜杠被破坏。
   - 如果直接渲染视频，先确认 `ffmpeg` 和 `ffprobe` 可用，或记录它们的显式路径。

4. 剪辑决策
   - 根据目标时长、画幅、脚本和素材角色生成时间线。
   - 每个片段必须有来源、入点、出点或时长、角色、可选字幕。
   - 计划里不得出现负时长、零时长、无来源片段或非预期空洞。

5. 输出执行
   - 计划 JSON：写入新输出目录。
   - 剪映/CapCut 草稿：生成新草稿或复制旧草稿后修改，默认不覆盖原草稿。
   - 直接渲染视频：使用项目渲染链路；没有时使用 `scripts/render_plan_ffmpeg.py`。
   - GUI 自动化：只有用户明确接受脆弱性时使用，并保留截图、日志和人工兜底步骤。

6. 验证闭环
   - 校验所有引用路径存在。
   - 校验计划 JSON 或 PRD JSON 可解析。
   - 校验总时长在容差内，片段时长全部为正。
   - 草稿输出必须能解析关键 JSON；视频输出必须存在、大小非零、可被 ffprobe 读取。
   - 生成验证报告，失败项不能包装成成功。

7. Ralph 迭代
   - 把需求拆成小 story，每个 story 都能独立验证。
   - 默认使用 `ralph-runner` dry-run + `--no-commit`。
   - 每轮 Ralph 输出后，读取 `.ralph/runs/`，把 story 状态、验证结果和日志路径反馈给用户。
   - 只有用户明确允许 commit，才运行正式提交模式。

## 推荐 Ralph Story 拆分

- `US-001` 需求文档与输出模式：创建或更新自动剪辑需求文档，明确输出模式、素材路径、目标时长、画幅、验收标准。
- `US-002` 素材 manifest：扫描输入目录，生成 manifest，记录存在状态和文件类型，不修改原素材。
- `US-003` 媒体元数据：探测视频/音频元数据，识别无音轨、混合画幅、可变帧率和旋转信息。
- `US-004` 剪辑计划：生成结构化时间线或 render-plan JSON，校验片段时长和总时长容差。
- `US-005` 输出生成：按输出模式生成方案、剪映/CapCut 新草稿或直接渲染 mp4。
- `US-006` 字幕与音频：处理 SRT 字幕、BGM、音量规则和必要的音轨兜底。
- `US-007` 验证报告：输出可解析报告，列出输入检查、时间线检查、产物检查、已运行命令和失败原因。

## Ralph 执行建议

默认项目别名优先使用 `li-auto-auto-cutting`。执行前必须先确认：

- 目标目录存在且是 Git 仓库。
- `git status --short` 干净。
- Ralph PRD 写入 `.agents/tasks/prd-auto-cutting.json` 或同类短 slug 路径。
- 首轮执行使用 `ralph.cmd build 1 --agent=codex --prd "<prd-path>" --no-commit`。

如果需求只是在完善文档，不要让 Ralph 直接实现渲染代码。如果用户明确要成片，Ralph story 必须包含渲染计划、执行命令和验证报告路径。

## 失败处理

- 素材缺失：停止输出执行，报告缺失路径。
- FFmpeg 不可用：保留 render-plan，报告安装或路径配置需求。
- 草稿 schema 不匹配：停止写入草稿，改为输出计划和差异说明。
- 验证失败：保留失败报告和日志路径，下一轮 Ralph 只处理失败 story。
